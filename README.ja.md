# canon-os

[English](README.md) | **日本語**

**コーディングエージェントのためのオペレーティングシステム。top-tier モデルの*振る舞い*を、支出ではなく構造で再現する。**

*作者: Sumika*

最も強力なモデルが従量課金制に移行しても、日常使いのエージェントには同じ規律で動いてほしいはずだ——結論ファースト、迷わず動く、
実際の出力で検証する、漏らさない、広げすぎない。canon-os は、mid-tier モデルを規律あるシニアのように*振る舞わせ*、そのモデルに
しかできない仕事のために高価な tier を温存する、プレーンテキストのカーネル + プレイブック + 振る舞いスタイルから成る小さな仕組みだ。

これは、誰でもインストールできるように切り出した、個人データを含まない汎用のプライベートエージェントOSのコアである。

## 主題: 3本柱 + 自己改善

品質を再現するのは、生の知能ではなく**構造**だ:

1. **構造** — 憲法、委任ポリシー、unknown-unknown 発見プロトコル、そしてローンチ用のプレイブック。多角的パネル + 敵対的統合 +
   カノン化。
2. **コンテキスト** — 出典レシート付きの二層構造(raw / compiled)ナレッジボールトと、索引優先の読み取り規律により、ナレッジベース
   は規模が大きくなるほど、ノイズではなく*強く*なる。
3. **振る舞い** — ツールのシステムプロンプト機構を通じて、top-tier モデルの「感触」(結論ファースト、動く、検証する、着地し直す)を
   どんなモデルにも注入する[振る舞いスタイル](behavior/operating-style.md)。
4. **自己改善** — 月次/四半期のメンテナンスレビュー、決定ログ、非推奨化ルール、そして unknowns 台帳により、システムは形骸化せず
   現実に即し続ける。

モデルの生の能力は、プロンプトでは転写されない。だからこそ、[委任ポリシー](kernel/delegation-policy.md)は最も難しい判断を top
tier に振り分け続ける——それ以外はすべて、規律正しく安価に動く。

## コンテキスト・エコノミー

コンテキストは当然与えられるものではなく、予算として扱う。すべてのワーカーに厳格な返却フォーマットと行数上限を課し、返ってくるの
が生のトランスクリプトではなくレポートになるようにする。まず索引に目を通し、実際に判断が必要になったときだけファイル全体を引く。
ワーカーのトランスクリプトを直接読み返すことはしない。決定は毎ターン、ファイルに書き出す——コンテキストは揮発性だが、ファイルは
コンパクションを生き延びる。全ルールセット: [playbooks/session-operations.md](playbooks/session-operations.md)。

## クイックスタート

```sh
git clone <this-repo> canon-os && cd canon-os && ./install.sh
```

または [INSTALL_PROMPT.md](INSTALL_PROMPT.md) をコーディングエージェントに貼り付け、自動的に組み込ませる。
全オプションと手動手順: [INSTALL.md](INSTALL.md)。tier とモデルの対応表: [MODELS.md](MODELS.md)。

**Claude Code**(output styles 経由)と **AGENTS.md 系ツール**(Codex 等)に対応。モデル非依存。

## 構成

- `kernel/` — 常時読み込まれる規範: [憲法](kernel/constitution.md)、
  [委任ポリシー](kernel/delegation-policy.md)、[セキュリティ・フロア](kernel/security-floor.md)、
  [ボールト・ポリシー](kernel/vault-policy.md)
- `behavior/` — [振る舞いスタイル](behavior/operating-style.md) + [サブエージェント前文](behavior/subagent-preamble.md)
- `playbooks/` — unknown-unknown プロトコル、プロジェクト起動、キックオフ、プロジェクト監査、メンテナンスレビュー、
  セッション運用
- `checklists/` — [バリューゲート](checklists/value-gate.md)
- `templates/` — プロジェクトエージェントファイル、実装ノート、進捗、引き継ぎ、憲章、決定、サブエージェント前文
- `maintenance/` — 非推奨化ルール + レビューログ・決定・バックログ用のスターター
- `INDEX.md` — 玄関口。最初に読むもの

## 謝辞

canon-os は、他者が公開したアイデアの上に成り立っている:

- 振る舞いスタイルの転写アプローチは、X 上の **@connect24h** による公開記事、および
  **Claude Code の output-styles** ドキュメント(code.claude.com/docs)を参考にしている。
- 二層構造のナレッジボールト・パターンは、X 上で **Machina(@EXM7777)** が広めた「セカンドブレイン」パターンの
  公開記事を参考にしている。
- 委任のヒント——強いモデルが定型作業を安いモデルへ振り分ける——は、**Simon Willison** のエージェント運用に関する
  記事に通じるものがある。

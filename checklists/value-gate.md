# Value Gate

Last reviewed: 2026-07-06 / Index: [INDEX.md](../INDEX.md)

Run this before committing to a project or a significant feature (Phase 0 of
[project-launch.md](../playbooks/project-launch.md)). It is a Go / No-Go on whether the thing is worth building.
A universal product-value check — no specific pricing philosophy assumed.

## A. Define the change

- Who is the user? Who is explicitly *not* the user for this?
- Before state → After state. Describe the change across as many axes as apply: capability, emotion, time saved,
  risk removed, relationship.
- State the change in one sentence. If you can't, don't build it (Constitution Article 6).

## B. Willingness to pay / worth

- What is this change worth to the user? Evidence: comparison to alternatives, the user's own words, real usage.
- Are you pricing (or budgeting effort) from the value delivered, or just from what it costs to make?

## C. Journey

- Awareness → consideration → decision → first use → habit → return/renew → referral. Can you name one drop-off
  reason at each stage?
- Is measurement instrumented from day one? (Data you don't capture, you can't recover.)

## D. Adoption

- Time to first value ("aha"). Friction at onboarding. Who is the counterpart on the user's side?

## E. Retention / after-care

- Is the reason to continue relationship or inertia? Is there a designed monthly touchpoint?
- Is a graduation condition defined (the user becoming self-sufficient)?

## F. What to cut

- Did you write a first "won't-build list"?
- Is there a criterion for declining a low-fit request or feature?

## G. Verdict

- If three or more of A–F are unanswerable → go back to discovery or the
  [unknown-unknown-protocol.md](../playbooks/unknown-unknown-protocol.md). Once filled, transcribe into the
  [charter](../templates/charter.template.md).
